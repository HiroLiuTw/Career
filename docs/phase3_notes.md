# Phase 3 Notes - Company-based Interview Workspace

## Change summary

This phase reorganizes interview preparation around individual companies instead of a broad knowledge-base hierarchy.

## Decision

Use `interview/<Company>/` as the main preparation unit.

Each company folder should contain:

- README.md
- company.md
- job_description.md
- technical_questions.md
- manager_questions.md
- my_answers.md
- feedback.md
- assets/

## Rationale

The main goal is job search execution, not building a general textbook-style knowledge base. Company-based folders make it easier to prepare, review, and update interview materials according to actual interview needs.

## Zyxel focus

Zyxel preparation emphasizes:

- C language fundamentals
- Linux userspace / kernel-space concepts
- TCP/IP, bridge, routing, VLAN, NAT, ARP
- Driver and module integration positioning
- DevicePort networking story
- Trend Micro production debugging and integration story
