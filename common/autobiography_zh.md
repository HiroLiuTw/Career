關於我
我是劉博文（Hiro Liu），在軟體業工作了超過十三年。我不是那種從一開始就鎖定單一領域的人，職涯一路走來接觸過工業設備、行動平台、端點資安，每一段都讓我對「程式怎麼跑在真實世界裡」有更深的體感。


從工業設備開始
剛出社會的時候，我加入了一家做工業用網路設備的公司。他們的產品是讓傳統的 Serial、Parallel 這類介面能透過網路延伸出去，用在 POS 機、工廠設備、IoT 場景。

一開始我做的是 Windows 的設備管理軟體，那個時期讓我對作業系統怎麼運作有了很紮實的認識,

後來公司推出了進階版的遠端連線產品，我的工作範疇一下子大了很多。除了重新設計通訊協定，我還同時開發了 Android 跟 iOS 的 SDK，讓其他廠商可以把我們的產品整合進自己的 App 裡。

再後來我做了一個實驗性質的專案，把整個設備管理的介面包成 RESTful API，用 Node.js 跑在 Docker 容器裡。那時候容器化還不像現在這麼普及，算是自己摸索著做出來的。

在這家公司待了幾年後，我開始帶一個三人小組，負責的除了自己的程式，還有帶人、排工作優先順序、跟其他部門協調進度。這段時間讓我理解到，帶人最難的不是技術，是怎麼讓大家對同一件事有一樣的認知。


加入趨勢科技
後來我進了趨勢科技，負責 Worry-Free Business Security 這個端點防護產品。這是一個全球跑在超過四百萬台裝置上的 SaaS 服務，規模跟之前完全不同。

在 Android 這塊，我維護的是直接裝在使用者手機上的 Security Agent，核心的掃描引擎是用 C/C++ 透過 NDK/JNI 接進來的。我做過最有成就感的事之一，是從零把自動化迴歸測試建起來——之前都是人工跑，我用 Appium 加 Python 寫了一套模擬真實操作的測試框架，之後每次發版前都能自動跑過一輪，省掉大量手動確認的時間。

Windows 這邊則是另一種挑戰。C++ Agent 部署在規模最大的一群機器上，問題也最複雜——system hang、效能下降、跟第三方軟體打架，這些都是很難在自己的環境重現的問題。我花了很多時間學會用 WinDBG、Process Monitor、Windows Performance Analyzer 這些工具，從 crash dump 或 trace log 裡找出根本原因。

客訴處理是這份工作很現實的一面。每年大概要消化七八十件案件，少數幾個 P0/P1 的緊急案子得直接遠端連進客戶的機器去 debug，有時候問題很明確，有時候得跟客戶來回幾輪才能重現。這個過程練出了我在資訊不完整的情況下快速做判斷的能力。

除了產品本身，我也蠻投入在工程流程這塊。CI/CD pipeline 的維護、用 AWS Athena 查 telemetry 資料找異常趨勢、在 Kibana 上設監控，這些讓我習慣用數據來理解產品狀況，而不是等問題爆出來再去救火。最近幾年公司開始大量導入 AI 工具，我自己也在用 Claude Code 幫日常的開發跟客訴分析提速，用起來比想像中有效。
